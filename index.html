Learn Hub: Core OS Scheduling Concepts

Completion Time (CT)

The time instant when a process finishes execution.

CT = Time of termination

Turnaround Time (TAT)

The total time spent by a process in the system from arrival to completion.

TAT = CT - Arrival Time

Waiting Time (WT)

The total time spent by a process waiting in the ready queue.

WT = TAT - Burst Time

Response Time (RT)

The time from arrival to the first time the process gets scheduled on the CPU.

RT = First Run Time - Arrival Time

1. First-Come First-Served (FCFS)

Simple FIFO queue logic. Simplest scheduler, non-preemptive. High potential for Convoy Effect.

2. Shortest Job First (SJF)

Selects process with shortest burst. Mathematically optimal for minimizing average waiting times. Hard to predict burst time in practice.

3. Shortest Remaining Time First (SRTF)

Preemptive version of SJF. Current task is preempted if a new process arrives with a shorter remaining run time.

4. Round Robin (RR)

Each process gets a slice of time (Quantum), then gets rotated back to queue. Prevents starvation, but introduces context switch overhead.

5. Priority Scheduling (NP / Preemptive)

Schedules based on numeric priorities (here, lower = higher priority). In preemptive mode, high priority tasks interrupt running processes instantly.

The Convoy Effect (FCFS)

Occurs when a single CPU-bound process with a huge burst time runs first, forcing multiple short I/O-bound processes to wait. CPU utilization drops since I/O devices sit idle.
Try the **FCFS Convoy Effect** preset to see this visually!

Process Starvation (SJF / SRTF / Priority)

A scheduling anomaly where a process waits indefinitely because other processes with higher priority (or shorter burst times) keep arriving and taking over the CPU.
Try the **SRTF Starvation Demo** preset to see P1 starve.

What is Aging?

Aging is a technique used to avoid **Starvation** in priority-based scheduling systems. It works by gradually increasing the priority of processes that wait in the ready queue for a long time.

For example, if a process has a starting low priority of `5`, and we set the aging threshold to `5s`, every `5s` the process spends waiting in the Ready queue, its priority decreases by `1` (which increases its priority level: `5 -> 4 -> 3 -> 2 -> 1`).

Eventually, the waiting process becomes the highest priority task in the queue, forcing the CPU to run it and solving starvation!
